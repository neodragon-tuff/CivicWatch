# 🗺️ CivicPulse Roadmap

## Phase 1 — Foundation (June 2026)
**Goal:** Working front-end dashboard, publicly deployed, good looking, actually useful

- [x] Core HTML/CSS/JS dashboard
- [x] Vote tracker with status and expandable details
- [x] Budget visualizer with animated breakdowns
- [x] Public meeting calendar with RSVP
- [x] Infrastructure status board with alerts
- [x] Multi-city support (6 cities)
- [x] Live news ticker
- [x] Toast notification system
- [x] Responsive mobile layout
- [x] GitHub Pages deployment
- [x] MIT License
- [x] Contributing guide
- [x] Developer journal started
- [ ] Keyboard navigation (ARIA)
- [ ] Unit tests for core render functions
- [ ] Open Graph meta tags for social sharing

## Phase 2 — Real Data (July–August 2026)
**Goal:** Replace hardcoded data with live city API calls

- [ ] Node.js + Express backend
- [ ] Connect to Boston Open Data API
- [ ] Connect to NYC Open Data API
- [ ] Data normalization layer (map different API shapes to our schema)
- [ ] 15-minute polling with caching
- [ ] Email alert subscriptions (SendGrid)
- [ ] PostgreSQL database for user RSVPs

## Phase 3 — Community (September–October 2026)
**Goal:** Let residents interact, not just observe

- [ ] User accounts (email only, no social login)
- [ ] Comment threads on votes
- [ ] Neighborhood-level filtering
- [ ] "Impact on me" plain-language summaries using AI
- [ ] Petition creation linked to live votes
- [ ] Accessibility audit + full screen reader support

## Phase 4 — Scale (2027)
**Goal:** National coverage and media partnerships

- [ ] 50-city support
- [ ] React Native mobile app
- [ ] Embed widget for local news outlets
- [ ] Journalist data export (CSV, JSON)
- [ ] Partnership with open data organizations
- [ ] Multi-language support (Spanish, Mandarin, Portuguese)
